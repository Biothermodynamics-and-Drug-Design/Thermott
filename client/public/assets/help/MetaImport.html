  • This is the most simplified version of metadata formats
  • Here you do not have to specifity binding "experiment_id"
  • The tool will automaticaly determine your dilution number and assign wells to the unique binding experiments

    • Metadata format Nº1
      • Each line represent a consecutive well
      • Binding experiment ordering is important
      • Each line should have comma separated:
        1. protein name - e.g. Carbonic anhydrase I
        2. protein concentration (μM) - e.g. 4
        3. ligand name - e.g. GZ17-02
        4. ligand concentration (μM) - e.g. 1000

  • Click here to download raw data file suitable for this metadata

  • This is recommended metadata format for conventional binding experiments

    • Metadata format Nº2
      • Each line represent a consecutive well.
      • The first column's "experiment_id" value assigns a single well to the binding experiment
      • Each line should have comma separated:
        1. experiment_id - numeric identifier of binding experiment e.g. 1
        2. experiment_name - binding experiment name e.g. "Binding experiment 1"
        3. protein name - e.g. Carbonic anhydrase I
        4. protein concentration (μM) - e.g. 4
        5. ligand name - e.g. GZ17-02
        6. ligand concentration (μM) - e.g. 1000

  • This metadata format is suitable for binding experiments that possess the same control

    • Metadata format Nº3
      • Each line represent a consecutive well.
      • The first column's "experiment_id" value assigns a single well to the binding experiment
      • The control wells must have 0 in the "experiment_id" column
      • Each line should have comma separated:
        1. experiment_id - numeric identifier of binding experiment e.g. 1
        2. experiment_name - binding experiment name e.g. "Binding experiment 1"
        3. protein name - e.g. Hsp90aN
        4. protein concentration (μM) - e.g. 10
        5. ligand name - e.g. IV11
        6. ligand concentration (μM) - e.g. 200
